What motivates a transfer: player performance, transfer fee or the rivalry between the teams?

Authors

Vignesh Thiruvel Selvam

Prof. Dr.Christoph Ihl

In order to answer this question, we gather the list of teams in the Top 5 leagues in Europe, and their transfer history (only players bought) from the Summer transfer window in 2018 to 2024. The league table (from the end of 2017/18 season to the end of 2024/25 season) is used to get the points that a team has scored at the end of a season and combined with ELO, is used to define a rivalry score, which is used in the analysis to check if teams tend to buy a player who is from a rival. We also collect a player’s attributes such as their age, nationality, minutes played, etc,. Combining these with the rivalry score and the transfer fee, we try to identify which of these variables primarily motivate a transfer.

Rivalry

Rivalry is defined based on the following strategy:

  1. We already have the league table history for teams. We now get the ELO rating of teams for all these seasons.
  2. Next, we define a vector: <points scored by a team this season, points scored by the team previous season, points scored by the team 2 seasons ago, current ELO>.
  3. Based on this, we calculate distances between teams. Note: We ignore missing data altogether. We have 2 such cases:
    1. We only have the data starting from the end of the 2017/18 season. Thus, for this season, the 2nd and 3rd values in the vector are not present. We leave this as is.
    2. Also, we could have missing data for a team that was newly promoted to the league. We again ignore missing values in the vector.
  4. Next, we attribute this distance score over multiple seasons to smoothen out the rivalry score, so that there aren’t drastic changes in rivalry across seasons.
  5. Next, we calculate a continuous rivalry proximity score, a rank-based percentile of the smoothened distance. Then, we invert it so that when the score is closer to 1, then the 2 teams are closer rivals.

fetch_elo_for_date <- function(date_str) {
  destfile <- file.path(out_elo_raw_dir, paste0("clubelo_", date_str, ".csv"))
  if (file.exists(destfile)) {
    message("Using cached Elo snapshot: ", destfile)
    df <- read_csv(destfile, show_col_types = FALSE)
    setDT(df); return(df)
  }
  url <- paste0("http://api.clubelo.com/", date_str)
  message("Fetching from API: ", url)
  r <- httr::GET(url)
  if (httr::status_code(r) != 200) stop("ClubElo request failed for date ", date_str, " (status ", httr::status_code(r), ")")
  txt <- httr::content(r, as = "text", encoding = "UTF-8")
  df <- read_csv(txt, show_col_types = FALSE)
  setDT(df)
  setnames(df, tolower(names(df)))
  club_col <- grep("club|team|name", names(df), value = TRUE)[1]
  elo_col  <- grep("elo|rating", names(df), value = TRUE)[1]
  if (is.na(club_col) || is.na(elo_col)) stop("Unable to find club/elo columns in ClubElo CSV for date: ", date_str)
  df <- df[, .(club = get(club_col), elo = as.numeric(get(elo_col)))]
  df <- df[!is.na(elo) & !is.na(club)]
  df[, club_clean := tolower(str_squish(gsub("\\(.*\\)", "", club)))]
  df <- df[order(-elo)]
  df[, snapshot_date := date_str]
  write_csv(df, destfile)
  message("Saved snapshot: ", destfile)
  return(df)
}

Calculating Rivalry score

Modified euclidean:

we only use the factors we have in the vector (pts_t, pts_t1, pts_t2, elo> to calculate distance after z score normalisation. Arbitrarily assigning 0 to missing values messed up the rivalry score calculation.

rivalry_scores <- rival_vectors[
  ,
  {
    X <- as.matrix(.SD)
    teams <- team_clean
    n <- nrow(X)

    out <- vector("list", n*(n-1)/2)
    idx <- 1L

    for (i in 1:(n-1)) {
      for (j in (i+1):n) {

        xi <- X[i, ]
        xj <- X[j, ]

        # identify observed dimensions
        # ignore dimension if BOTH are NA (structurally missing)
        observed <- !(is.na(xi) & is.na(xj))

        k <- sum(observed)

        if (k == 0) {
          d <- NA_real_
        } else {
          d <- sqrt(sum((xi[observed] - xj[observed])^2) / k)
        }

        out[[idx]] <- list(
          team_1 = teams[i],
          team_2 = teams[j],
          rivalry_score = d
        )
        idx <- idx + 1L
      }
    }

    rbindlist(out)
  },
  by = .(country, season_start),
  .SDcols = c("z_pts_t", "z_pts_t1", "z_pts_t2", "z_elo")

Transfers

We now get the transfer data for all teams in the top 5 leagues from 2018 to 2024 summer transfer windows. We only fetch the summer window transfers. We don’t consider winter transfers for our models because:

  1. Winter transfers are based on a team’s position in the middle of a season. Often, it is considered to be a temporary solution (to fill up an injured player’s spot or a desperate transfer to avoid relegation, etc.,.).
  2. It could even affect a team’s options in the next summer window, due to constrained budget (already spent on players in the winter transfer), overcrowded positions (a player was bought due to an injury to an existing player, but the injured player has now returned, etc.).

Thus, it is altogether a different strategy compared to summer transfers and beyond the scope of what we are trying to establish in this analysis.

# ---------- HELPERS ----------
safe_fn <- function(x) gsub("[^A-Za-z0-9_\\-]", "_", substr(x, 1, 120))

league_urls_cache_file <- function(country_name, start_year) {
  file.path(cache_dir, paste0("league_urls_", safe_fn(country_name), "_", start_year, ".rds"))
}
team_transfers_cache_file <- function(team_url, season_start) {
  file.path(cache_dir, paste0("team_transfers_", safe_fn(team_url), "_", season_start, ".rds"))
}

# cached call for league -> team urls
get_league_team_urls_cached <- function(country_name, start_year) {
  cachef <- league_urls_cache_file(country_name, start_year)
  if (file.exists(cachef)) return(readRDS(cachef))
  for (i in seq_len(max_retries)) {
    try({
      urls <- tm_league_team_urls(country_name = country_name, start_year = start_year)
      saveRDS(urls, cachef)
      Sys.sleep(sleep_sec)
      return(urls)
    }, silent = TRUE)
    message("Retrying league-team-urls for ", country_name, " ", start_year, " (attempt ", i, ")")
    Sys.sleep(retry_sleep)
  }
  warning("Failed to fetch league team URLs for ", country_name, " ", start_year)
  return(NULL)
}

# cached call for team -> transfers (summer)
get_team_transfers_cached <- function(team_url, season_start) {
  cachef <- team_transfers_cache_file(team_url, season_start)
  if (file.exists(cachef)) return(readRDS(cachef))
  for (i in seq_len(max_retries)) {
    try({
      xf <- tm_team_transfers(team_url = team_url, transfer_window = "summer")
      saveRDS(xf, cachef)
      Sys.sleep(sleep_sec)
      return(xf)
    }, silent = TRUE)
    message("Retrying team transfers for ", team_url, " (attempt ", i, ")")
    Sys.sleep(retry_sleep)
  }
  message("No transfers found for team: ", team_url)
  return(NULL)
}

# cheap fuzzy compare to decide whether a returned team name matches a squad in your league table
close_name_match <- function(x, candidates, max_dist = 0.18) {
  # x: single string, candidates: vector of strings
  if (is.na(x) || x == "") return(FALSE)
  # use jaro-winkler via stringdist if available; fallback to grepl substring match
  if (requireNamespace("stringdist", quietly = TRUE)) {
    d <- stringdist::stringdist(tolower(str_squish(x)), tolower(str_squish(candidates)), method = "jw")
    any(d <= max_dist)
  } else {
    any(grepl(tolower(str_squish(x)), tolower(candidates), fixed = TRUE))
  }
}

    }

Interpretation

Model 1 and model 2 provided different outputs for rivalry because in model 1, alternatives mostly come from outside the league. So, rivalry_proximity was mostly 0 and it had weak identification. Model 2 included players sold from the current league as well, so the rivalry proximity wasn’t all 0s, leading to a much stronger identification. Also, based on both concordance and LR test, our second model fits the data significantly better.

Conclusion

The results show that the effect of rivalry depends strongly on the set of feasible transfer alternatives. When the choice set is restricted to incoming transfers (Model 1), rivalry proximity has a positive but statistically insignificant effect. However, when the alternative set is expanded to include players leaving the focal league (Model 2), rivalry proximity becomes strongly negative and highly significant. This suggests that clubs are less likely to acquire players from rival teams when realistic within-league alternatives are available. Additionally, clubs prefer if a player has slightly more experience and is one of their own.
